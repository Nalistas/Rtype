template<typename Function >
void ecs::registry::add_standalone_system ( Function &&  f)

Add a system that does not depend on any components. This version is used for systems that operate independently of specific components in the registry.

Template Parameters
FunctionThe type of the system (should inherit from ecs::isystem<> or be a compatible callable). The system must implement void operator()(iregistry &) const.
Parameters
fThe system to add. It is a callable that takes only a reference to the registry.
Note
This function is intended for global systems or those that do not process entity components.

◆ add_system()

template<class... Components, typename Function >
void ecs::registry::add_system ( Function &&  f)

Add a system to the registry that operates on specific components. This version is used for systems that process entities with specific components.

Template Parameters
ComponentsThe components used by the system.
FunctionThe type of the system (should inherit from ecs::isystem<Components...>). The system must implement void operator()(iregistry &, Components& ...) const.
Parameters
fThe system to add. It is a callable that takes a reference to the registry and the required components as parameters.
Note
Requires the specified components to be present in the sparse arrays of the registry. This function will encapsulate the system call to process entities with matching components.

This package contains two extensions that add support for MDX ESM syntax in markdown to \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}}. These extensions plug into \href{https://github.com/syntax-tree/mdast-util-from-markdown}{\texttt{ {\ttfamily mdast-\/util-\/from-\/markdown}}} (to support parsing ESM in markdown into a syntax tree) and \href{https://github.com/syntax-tree/mdast-util-to-markdown}{\texttt{ {\ttfamily mdast-\/util-\/to-\/markdown}}} (to support serializing ESM in syntax trees to markdown).\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-mdxjs-esm_2readme_autotoc_md3939}{}\doxysection{\texorpdfstring{When to use this}{When to use this}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-mdxjs-esm_2readme_autotoc_md3939}
You can use these extensions when you are working with {\ttfamily mdast-\/util-\/from-\/markdown} and {\ttfamily mdast-\/util-\/to-\/markdown} already.

When working with {\ttfamily mdast-\/util-\/from-\/markdown}, you must combine this package with \href{https://github.com/micromark/micromark-extension-mdxjs-esm}{\texttt{ {\ttfamily micromark-\/extension-\/mdxjs-\/esm}}}.

When you are working with syntax trees and want all of MDX, use \href{https://github.com/syntax-tree/mdast-util-mdx}{\texttt{ {\ttfamily mdast-\/util-\/mdx}}} instead.

All these packages are used in \href{https://mdxjs.com/packages/remark-mdx/}{\texttt{ {\ttfamily remark-\/mdx}}}, which focusses on making it easier to transform content by abstracting these internals away.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-mdxjs-esm_2readme_autotoc_md3940}{}\doxysection{\texorpdfstring{Install}{Install}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-mdxjs-esm_2readme_autotoc_md3940}

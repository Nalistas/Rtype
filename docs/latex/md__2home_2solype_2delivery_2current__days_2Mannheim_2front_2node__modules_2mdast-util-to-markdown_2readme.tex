This package is a utility that takes an \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}} syntax tree as input and turns it into serialized markdown.

This utility is a low level project. It’s used in \href{https://github.com/remarkjs/remark/tree/main/packages/remark-stringify}{\texttt{ {\ttfamily remark-\/stringify}}}, which focusses on making it easier to transform content by abstracting these internals away.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-to-markdown_2readme_autotoc_md4013}{}\doxysection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-to-markdown_2readme_autotoc_md4013}
If you want to handle syntax trees manually, use this. For an easier time processing content, use the {\bfseries{\href{https://github.com/remarkjs/remark}{\texttt{ remark}}}} ecosystem instead.

You can combine this utility with other utilities to add syntax extensions. Notable examples that deeply integrate with it are \href{https://github.com/syntax-tree/mdast-util-gfm}{\texttt{ {\ttfamily mdast-\/util-\/gfm}}}, \href{https://github.com/syntax-tree/mdast-util-mdx}{\texttt{ {\ttfamily mdast-\/util-\/mdx}}}, \href{https://github.com/syntax-tree/mdast-util-frontmatter}{\texttt{ {\ttfamily mdast-\/util-\/frontmatter}}}, \href{https://github.com/syntax-tree/mdast-util-math}{\texttt{ {\ttfamily mdast-\/util-\/math}}}, and \href{https://github.com/syntax-tree/mdast-util-directive}{\texttt{ {\ttfamily mdast-\/util-\/directive}}}.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-to-markdown_2readme_autotoc_md4014}{}\doxysection{\texorpdfstring{Install}{Install}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-to-markdown_2readme_autotoc_md4014}

{\ttfamily }\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-to-markdown_2readme_autotoc_md4037}{}\doxysection{\texorpdfstring{{\ttfamily Security}}{{\ttfamily Security}}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2mdast-util-to-markdown_2readme_autotoc_md4037}
{\ttfamily }

{\ttfamily {\ttfamily mdast-\/util-\/to-\/markdown} will do its best to serialize markdown to match the syntax tree, but there are several cases where that is impossible. It’ll do its best, but complete roundtripping is impossible given that any value could be injected into the tree.}

{\ttfamily As markdown is sometimes used for HTML, and improper use of HTML can open you up to a \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attack, use of {\ttfamily mdast-\/util-\/to-\/markdown} and parsing it again later could potentially be unsafe. When parsing markdown afterwards and then going to HTML, use something like \href{https://github.com/syntax-tree/hast-util-sanitize}{\texttt{ {\ttfamily hast-\/util-\/sanitize}}} to make the tree safe.}

Append {\ttfamily items} (an array) at the end of {\ttfamily list} (another array). When {\ttfamily list} was empty, returns {\ttfamily items} instead.

This prevents a potentially expensive operation when {\ttfamily list} is empty, and adds items in batches to prevent V8 from hanging.

Either {\ttfamily list} or {\ttfamily items} ({\ttfamily Array\texorpdfstring{$<$}{<}unknown\texorpdfstring{$>$}{>}}).\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2micromark-util-chunked_2readme_autotoc_md4180}{}\doxysubsection{\texorpdfstring{{\ttfamily splice(list, start, remove, items)}}{{\ttfamily splice(list, start, remove, items)}}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2micromark-util-chunked_2readme_autotoc_md4180}
Like {\ttfamily Array\#splice}, but smarter for giant arrays.

{\ttfamily Array\#splice} takes all items to be inserted as individual argument which causes a stack overflow in V8 when trying to insert 100k items for instance.

Otherwise, this does not return the removed items, and takes {\ttfamily items} as an array instead of rest parameters.

\item {\ttfamily remove} ({\ttfamily number}) — number of items to remove
\item {\ttfamily items} ({\ttfamily Array\texorpdfstring{$<$}{<}unknown\texorpdfstring{$>$}{>}}) — items to inject into {\ttfamily list}
\end{DoxyItemize}

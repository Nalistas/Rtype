This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/remarkjs/remark}{\texttt{ remark}}) plugin that defines how to take markdown as input and turn it into a syntax tree.

See \href{https://github.com/remarkjs/remark}{\texttt{ the monorepo readme}} for info on what the remark ecosystem is.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2remark-parse_2readme_autotoc_md5492}{}\doxysection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2remark-parse_2readme_autotoc_md5492}
This plugin adds support to unified for parsing markdown. If you also need to serialize markdown, you can alternatively use \href{../remark/}{\texttt{ {\ttfamily remark}}}, which combines unified, this plugin, and \href{../remark-stringify/}{\texttt{ {\ttfamily remark-\/stringify}}}.

If you {\itshape just} want to turn markdown into HTML (with maybe a few extensions), we recommend \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}} instead. If you don’t use plugins and want to access the syntax tree, you can directly use \href{https://github.com/syntax-tree/mdast-util-from-markdown}{\texttt{ {\ttfamily mdast-\/util-\/from-\/markdown}}}. remark focusses on making it easier to transform content by abstracting these internals away.

You can combine this plugin with other plugins to add syntax extensions. Notable examples that deeply integrate with it are \href{https://github.com/remarkjs/remark-gfm}{\texttt{ {\ttfamily remark-\/gfm}}}, \href{https://github.com/mdx-js/mdx/tree/main/packages/remark-mdx}{\texttt{ {\ttfamily remark-\/mdx}}}, \href{https://github.com/remarkjs/remark-frontmatter}{\texttt{ {\ttfamily remark-\/frontmatter}}}, \href{https://github.com/remarkjs/remark-math}{\texttt{ {\ttfamily remark-\/math}}}, and \href{https://github.com/remarkjs/remark-directive}{\texttt{ {\ttfamily remark-\/directive}}}. You can also use any other \href{https://github.com/remarkjs/remark\#plugin}{\texttt{ remark plugin}} after {\ttfamily remark-\/parse}.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2remark-parse_2readme_autotoc_md5493}{}\doxysection{\texorpdfstring{Install}{Install}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2remark-parse_2readme_autotoc_md5493}

Man pages (short for manual pages) are a way to document CLIs (example\+: type {\ttfamily man git-\/log} in your terminal). They use an old markup format called roff. There’s a remark plugin, \href{https://github.com/remarkjs/remark-man}{\texttt{ {\ttfamily remark-\/man}}}, that can serialize as roff.

This example shows how to turn markdown into man pages by using unified with {\ttfamily remark-\/parse} and {\ttfamily remark-\/man}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ remarkMan\ from\ 'remark-\/man'}
\DoxyCodeLine{import\ remarkParse\ from\ 'remark-\/parse'}
\DoxyCodeLine{import\ \{unified\}\ from\ 'unified'}
\DoxyCodeLine{}
\DoxyCodeLine{const\ doc\ =\ \`{}}
\DoxyCodeLine{\#\ titan(7)\ -\/-\/\ largest\ moon\ of\ saturn}
\DoxyCodeLine{}
\DoxyCodeLine{Titan\ is\ the\ largest\ moon…}
\DoxyCodeLine{\`{}}
\DoxyCodeLine{}
\DoxyCodeLine{const\ file\ =\ await\ unified().use(remarkParse).use(remarkMan).process(doc)}
\DoxyCodeLine{}
\DoxyCodeLine{console.log(String(file))}

\end{DoxyCode}

\DoxyCodeLine{.P}
\DoxyCodeLine{Titan\ is\ the\ largest\ moon…}

Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily remark-\/parse@\texorpdfstring{$^\wedge$}{\string^}11}, compatible with Node.\+js 16.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2remark-parse_2readme_autotoc_md5504}{}\doxysection{\texorpdfstring{Security}{Security}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2remark-parse_2readme_autotoc_md5504}
As markdown can be turned into HTML and improper use of HTML can open you up to \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attacks, use of remark can be unsafe. When going to HTML, you will combine remark with {\bfseries{\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}}}, in which case you should use \href{https://github.com/rehypejs/rehype-sanitize}{\texttt{ {\ttfamily rehype-\/sanitize}}}.

Use of remark plugins could also open you up to other attacks. Carefully assess each plugin and the risks involved in using them.

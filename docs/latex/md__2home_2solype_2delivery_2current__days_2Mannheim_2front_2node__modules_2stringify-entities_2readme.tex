This is a small and powerful encoder of HTML character references (often called entities). This one has either all the options you need for a minifier/formatter, or a tiny size when using {\ttfamily stringify\+Entities\+Light}.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2stringify-entities_2readme_autotoc_md5767}{}\doxysection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2stringify-entities_2readme_autotoc_md5767}
You can use this for spec-\/compliant encoding of character references. It‚Äôs small and fast enough to do that well. You can also use this when making an HTML formatter or minifier, because there are different ways to produce pretty or tiny output. This package is reliable\+: {\ttfamily \textquotesingle{}\`{}\textquotesingle{}} characters are encoded to ensure no scripts run in Internet Explorer 6 to 8. Additionally, only named references recognized by HTML 4 are encoded, meaning the infamous {\ttfamily \&apos;} (which people think is a \href{https://www.telegraph.co.uk/technology/advice/10516839/Why-do-some-apostrophes-get-replaced-with-andapos.html}{\texttt{ virus}}) won‚Äôt show up.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2stringify-entities_2readme_autotoc_md5768}{}\doxysection{\texorpdfstring{Install}{Install}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2stringify-entities_2readme_autotoc_md5768}

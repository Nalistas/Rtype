Object type/kind confirmation, returns either {\ttfamily true} or {\ttfamily false}.

 \doxysubparagraph*{{\ttfamily \texorpdfstring{$\ast$}{*}/ensure}}

Value validation. Returns input value (in primitive cases possibly coerced) or if value doesn\textquotesingle{}t meet the constraints throws {\ttfamily Type\+Error} .

Each {\ttfamily \texorpdfstring{$\ast$}{*}/ensure} utility, accepts following options (eventually passed with second argument)\+:


\begin{DoxyItemize}
\item {\ttfamily is\+Optional} -\/ Makes {\ttfamily null} or {\ttfamily undefined} accepted as valid value. In such case instead of {\ttfamily Type\+Error} being thrown, {\ttfamily null} is returned.
\item {\ttfamily default} -\/ A value to be returned if {\ttfamily null} or {\ttfamily undefined} is passed as an input value.
\item {\ttfamily error\+Message} -\/ Custom error message. Following placeholders can be used\+:
\begin{DoxyItemize}
\item {\ttfamily v} -\/ To be replaced with short string representation of invalid value
\item {\ttfamily n} -\/ To be replaced with meaninfgul name (to be passed with {\ttfamily name} option) of validated value. Not effective if {\ttfamily name} option is not present
\end{DoxyItemize}
\item {\ttfamily error\+Code} -\/ Eventual error code to be exposed on {\ttfamily .code} error property
\item {\ttfamily name} -\/ Meaningful name for validated value, to be used in error message, assuming it contains {\ttfamily n} placeholder
\item {\ttfamily Error} -\/ Alternative error constructor to be used (defaults to {\ttfamily Type\+Error})

This utility helps with accessing positional info on a potentially dirty tree.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2unist-util-position_2readme_autotoc_md6788}{}\doxysection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2unist-util-position_2readme_autotoc_md6788}
The positional info is typically consistent and proper in unist trees generated by our ecosystem, but, user plugins could mess that up. If you’re making a reusable plugin, and accessing the positional info often, you might want to guard against that by using this utility.

You might also find the utility \href{https://github.com/syntax-tree/unist-util-generated}{\texttt{ {\ttfamily unist-\/util-\/generated}}} useful to check whether a node is considered to be generated (not in the original input file).

You might also enjoy \href{https://github.com/syntax-tree/unist-util-stringify-position}{\texttt{ {\ttfamily unist-\/util-\/stringify-\/position}}} when you want to display positional info to users.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2unist-util-position_2readme_autotoc_md6789}{}\doxysection{\texorpdfstring{Install}{Install}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2unist-util-position_2readme_autotoc_md6789}

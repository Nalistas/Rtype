This package provides a virtual file format. It exposes an API to access the file value, path, metadata about the file, and specifically supports attaching lint messages and errors to certain places in these files.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2vfile_2readme_autotoc_md6950}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2vfile_2readme_autotoc_md6950}
The virtual file format is useful when dealing with the concept of files in places where you might not be able to access the file system. The message API is particularly useful when making things that check files (as in, linting).

vfile is made for \href{https://github.com/unifiedjs/unified}{\texttt{ unified}}, which amongst other things checks files. However, vfile can be used in other projects that deal with parsing, transforming, and serializing data, to build linters, compilers, static site generators, and other build tools.

This is different from the excellent \href{https://github.com/gulpjs/vinyl}{\texttt{ {\ttfamily vinyl}}} in that vfile has a smaller API, a smaller size, and focuses on messages.\hypertarget{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2vfile_2readme_autotoc_md6951}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{md__2home_2solype_2delivery_2current__days_2Mannheim_2front_2node__modules_2vfile_2readme_autotoc_md6951}

Create a new virtual file.

{\ttfamily options} is treated as\+:


\begin{DoxyItemize}
\item {\ttfamily string} or \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Uint8Array}{\texttt{ {\ttfamily Uint8\+Array}}} — {\ttfamily \{value\+: options\}}
\item {\ttfamily URL} — {\ttfamily \{path\+: options\}}
\item {\ttfamily VFile} — shallow copies its data over to the new file
\item {\ttfamily object} — all fields are shallow copied over to the new file
\end{DoxyItemize}

Path related fields are set in the following order (least specific to most specific)\+: {\ttfamily history}, {\ttfamily path}, {\ttfamily basename}, {\ttfamily stem}, {\ttfamily extname}, {\ttfamily dirname}.

You cannot set {\ttfamily dirname} or {\ttfamily extname} without setting either {\ttfamily history}, {\ttfamily path}, {\ttfamily basename}, or {\ttfamily stem} too.

\item {\ttfamily stored} ({\ttfamily boolean}) — whether a file was saved to disk; this is used by vfile reporters
\end{DoxyItemize}

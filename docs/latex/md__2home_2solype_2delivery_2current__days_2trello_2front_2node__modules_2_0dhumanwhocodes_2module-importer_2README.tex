A utility for seamlessly importing modules in Node.\+js regardless if they are Common\+JS or ESM format. Under the hood, this uses {\ttfamily import()} and relies on Node.\+js\textquotesingle{}s Common\+JS compatibility to work correctly. This ensures that the correct locations and formats are used for Common\+JS so you can call one method and not worry about any compatibility issues.

The problem with the default {\ttfamily import()} is that it always resolves relative to the file location in which it is called. If you want to resolve from a different location, you need to jump through a few hoops to achieve that. This package makes it easy to both resolve and import modules from any directory.\hypertarget{md__2home_2solype_2delivery_2current__days_2trello_2front_2node__modules_2_0dhumanwhocodes_2module-importer_2README_autotoc_md7889}{}\doxysection{\texorpdfstring{Usage}{Usage}}\label{md__2home_2solype_2delivery_2current__days_2trello_2front_2node__modules_2_0dhumanwhocodes_2module-importer_2README_autotoc_md7889}

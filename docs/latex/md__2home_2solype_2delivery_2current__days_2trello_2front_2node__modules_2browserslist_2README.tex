The config to share target browsers and Node.\+js versions between different front-\/end tools. It is used in\+:


\begin{DoxyItemize}
\item \href{https://github.com/postcss/autoprefixer}{\texttt{ Autoprefixer}}
\item \href{https://github.com/babel/babel/tree/master/packages/babel-preset-env}{\texttt{ Babel}}
\item \href{https://github.com/csstools/postcss-plugins/tree/main/plugin-packs/postcss-preset-env}{\texttt{ postcss-\/preset-\/env}}
\item \href{https://github.com/amilajack/eslint-plugin-compat}{\texttt{ eslint-\/plugin-\/compat}}
\item \href{https://github.com/ismay/stylelint-no-unsupported-browser-features}{\texttt{ stylelint-\/no-\/unsupported-\/browser-\/features}}
\item \href{https://github.com/csstools/postcss-normalize}{\texttt{ postcss-\/normalize}}
\item \href{https://github.com/ElemeFE/obsolete-webpack-plugin}{\texttt{ obsolete-\/webpack-\/plugin}}
\end{DoxyItemize}

All tools will find target browsers automatically, when you add the following to {\ttfamily package.\+json}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}browserslist"{}:\ [}
\DoxyCodeLine{\ \ "{}defaults\ and\ fully\ supports\ es6-\/module"{},}
\DoxyCodeLine{\ \ "{}maintained\ node\ versions"{}}
\DoxyCodeLine{]}

\end{DoxyCode}


Or in {\ttfamily .browserslistrc} config\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\#\ Browsers\ that\ we\ support}
\DoxyCodeLine{}
\DoxyCodeLine{defaults\ and\ fully\ supports\ es6-\/module}
\DoxyCodeLine{maintained\ node\ versions}

\end{DoxyCode}


Developers set their version lists using queries like {\ttfamily last 2 versions} to be free from updating versions manually. Browserslist will use \href{https://github.com/ben-eb/caniuse-lite}{\texttt{ {\ttfamily caniuse-\/lite}}} with \href{https://caniuse.com/}{\texttt{ Can I Use}} data for this queries.

You can check how config works at our playground\+: \href{https://browsersl.ist/}{\texttt{ {\ttfamily browsersl.\+ist}}}

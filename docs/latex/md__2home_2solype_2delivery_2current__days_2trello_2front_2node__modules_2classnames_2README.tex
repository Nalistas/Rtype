The {\ttfamily class\+Names} function takes any number of arguments which can be a string or object. The argument `\textquotesingle{}foo'{\ttfamily is short for}\{ foo\+: true \}\`{}. If the value associated with a given key is falsy, that key won\textquotesingle{}t be included in the output.


\begin{DoxyCode}{0}
\DoxyCodeLine{classNames('foo',\ 'bar');\ //\ =>\ 'foo\ bar'}
\DoxyCodeLine{classNames('foo',\ \{\ bar:\ true\ \});\ //\ =>\ 'foo\ bar'}
\DoxyCodeLine{classNames(\{\ 'foo-\/bar':\ true\ \});\ //\ =>\ 'foo-\/bar'}
\DoxyCodeLine{classNames(\{\ 'foo-\/bar':\ false\ \});\ //\ =>\ ''}
\DoxyCodeLine{classNames(\{\ foo:\ true\ \},\ \{\ bar:\ true\ \});\ //\ =>\ 'foo\ bar'}
\DoxyCodeLine{classNames(\{\ foo:\ true,\ bar:\ true\ \});\ //\ =>\ 'foo\ bar'}
\DoxyCodeLine{}
\DoxyCodeLine{//\ lots\ of\ arguments\ of\ various\ types}
\DoxyCodeLine{classNames('foo',\ \{\ bar:\ true,\ duck:\ false\ \},\ 'baz',\ \{\ quux:\ true\ \});\ //\ =>\ 'foo\ bar\ baz\ quux'}
\DoxyCodeLine{}
\DoxyCodeLine{//\ other\ falsy\ values\ are\ just\ ignored}
\DoxyCodeLine{classNames(null,\ false,\ 'bar',\ undefined,\ 0,\ 1,\ \{\ baz:\ null\ \},\ '');\ //\ =>\ 'bar\ 1'}

\end{DoxyCode}


Arrays will be recursively flattened as per the rules above\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ arr\ =\ ['b',\ \{\ c:\ true,\ d:\ false\ \}];}
\DoxyCodeLine{classNames('a',\ arr);\ //\ =>\ 'a\ b\ c'}

\end{DoxyCode}

\hypertarget{md__2home_2solype_2delivery_2current__days_2trello_2front_2node__modules_2classnames_2README_autotoc_md9101}{}\doxysubsection{\texorpdfstring{Usage with React.\+js}{Usage with React.\+js}}\label{md__2home_2solype_2delivery_2current__days_2trello_2front_2node__modules_2classnames_2README_autotoc_md9101}
This package is the official replacement for {\ttfamily class\+Set}, which was originally shipped in the React.\+js Addons bundle.

One of its primary use cases is to make dynamic and conditional {\ttfamily class\+Name} props simpler to work with (especially more so than conditional string manipulation). So where you may have the following code to generate a {\ttfamily class\+Name} prop for a {\ttfamily \texorpdfstring{$<$}{<}button\texorpdfstring{$>$}{>}} in React\+:

\DoxyCodeLine{\ \ \ \ const\ [isHovered,\ setIsHovered]\ =\ useState(false);}
\DoxyCodeLine{}
\DoxyCodeLine{\ \ \ \ let\ btnClass\ =\ 'btn';}
\DoxyCodeLine{\ \ \ \ if\ (isPressed)\ btnClass\ +=\ '\ btn-\/pressed';}
\DoxyCodeLine{\ \ \ \ else\ if\ (isHovered)\ btnClass\ +=\ '\ btn-\/over';}

\DoxyCodeLine{\ \ \ \ return\ (}

\hypertarget{md__2home_2solype_2delivery_2current__days_2trello_2front_2node__modules_2classnames_2README_autotoc_md9102}{}\doxysubsection{\texorpdfstring{Alternate {\ttfamily dedupe} version}{Alternate {\ttfamily dedupe} version}}\label{md__2home_2solype_2delivery_2current__days_2trello_2front_2node__modules_2classnames_2README_autotoc_md9102}
There is an alternate version of {\ttfamily class\+Names} available which correctly dedupes classes and ensures that falsy classes specified in later arguments are excluded from the result set.

This version is slower (about 5x) so it is offered as an opt-\/in.

To use the dedupe version with Node.\+js, Browserify, or webpack\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ classNames\ =\ require('classnames/dedupe');}
\DoxyCodeLine{}
\DoxyCodeLine{classNames('foo',\ 'foo',\ 'bar');\ //\ =>\ 'foo\ bar'}
\DoxyCodeLine{classNames('foo',\ \{\ foo:\ false,\ bar:\ true\ \});\ //\ =>\ 'bar'}

\end{DoxyCode}

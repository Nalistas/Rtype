 Starting with v3.\+6.\+0, the CDN versions of date-\/fns are available on \href{https://www.jsdelivr.com/package/npm/date-fns}{\texttt{ js\+Delivr}} and other CDNs. They expose the date-\/fns functionality via the {\ttfamily window.\+date\+Fns} global variable.

Unlike the npm package, the CDN is transpiled to be compatible with IE11, so it supports a wide variety of legacy browsers and environments.

The CDN versions are available for the main module, all \& individual locales, and the FP submodule.

They come in two flavors\+: {\ttfamily cdn.\+js} and {\ttfamily cdn.\+min.\+js}. The latter is minified and should be used in production. The former is useful for debugging and development.

Keep in mind that using the CDN versions in production is suboptimal because they bundle all the date-\/fns functionality you will never use. It\textquotesingle{}s much better to use the npm package and a tree-\/shaking-\/enabled bundler like webpack or Rollup.
